学生分数的最小差值

给你一个 下标从 0 开始 的整数数组 nums ，其中 nums[i] 表示第 i 名学生的分数。另给你一个整数 k 。

从数组中选出任意 k 名学生的分数，使这 k 个分数间 最高分 和 最低分 的 差值 达到 最小化 。

返回可能的 最小差值 。

输出

分析

  • 第一种:
    • 暴力解法O(n^k^)直接忽略
  • 第二种:
    • 排序后用滑动窗口,最小值在窗口的左边,最大值在窗口右边,继续扩大的话最小值只会更大
    • 这k 个元素必然是有序数组中（排序后）的连续段

RedisServer命令

发表于 2022-02-10 分类于 Redis ， Server命令

RedisServer命令

库的数目

  • Redis默认有16个库

Server命令(列举一些常见命令,更多请参阅官方文档)

  • DBSIZE

    • 返回当前库的key的数目

  • FLUSHALL [ASYNC|SYNC]

    • 删除所有的库里面的key
    • 尽量避免使用FLUSHALL同步FLUSHALL会造成Redis阻塞,异步的也会造成Redis大量的消耗,增加Redis压力,生产环境中建议禁用该命令。

  • FLUSHDB [ASYNC|SYNC]

    • 删除当前库里面的所有Key
    • 同样禁止使用

  • INFO [section]

    • 返回服务器的有关信息
    • 可选参数如下:
      • server: 关于Redis服务器的一般信息
      • clients: 客户端连接部分
      • memory: 内存消耗相关信息
      • persistence: RDB和AOF相关信息
      • stats: 一般统计数据
      • replication: 主从复制相关信息
      • cpu: cpu消耗相关信息
      • commandstats: Redis命令相关统计
      • latencystats: Redis命令延迟百分比分布统计
      • cluster: Redis分片相关
      • modules: 模块相关
      • keyspac: 数据库相关
      • errorstats: Redis错误统计
      • all: Return all sections (excluding module generated ones)
      • default: Return only the default set of sections
      • everything: Includes all and modules

  • LASTSAVE

    • 返回Redis上一次备份的时间
    • 可以用来判断Redis备份是否正常

  • BGSAVE

    • 非阻塞的进行Redis保存

Redis介绍

发表于 2022-02-10 分类于 Redis

Redis介绍

Redis 是一个开源（BSD许可）的，基于内存的的数据结构存储系统，它可以用作数据库、缓存和消息中间件。 它支持多种类型的数据结构，如 字符串（strings），散列（hashes）， 列表（lists）， 集合（sets）， 有序集合（sorted sets） 与范围查询， bitmaps， hyperloglogs 和 地理空间(geospatial)索引半径查询。 Redis 内置了复制(replication)，LUA脚本(Lua scripting)，LRU驱动事件(LRU eviction)，事务(transactions)和不同级别的 磁盘持久化(persistence)， 并通过Redis哨兵(Sentinel) 和自动分区(Cluster)提供高可用性（high availability）

特点

  • 内存数据库，速度快，也支持数据的持久化，可以将内存中的数据保存在磁盘中，重启的时候可以再次加载进行使用。
  • Redis不仅仅支持简单的key-value类型的数据，同时还提供list，set，zset，hash等多种数据结构的存储。
  • Redis支持数据的备份/读写分离(主节点也是可以读的不过大部分操作都分配到从节点)(master-slave)与集群(分片存储)，以及拥有哨兵监控机制(主节点挂掉,立即选一个新的)。
  • 支持事务

优势

  • 性能极高 – Redis能读的速度是110000次/s，写的速度是81000次/s 。
  • 丰富的数据类型 – Redis支持 Strings、Lists、 Hashes、Sets 、Sorted Sets 等数据类型操作。
  • 原子操作 – Redis的所有操作都是原子性的，同时Redis还支持对几个操作合并后的原子性执行。（事务）
  • 丰富的特性 – Redis还支持 publish/subscribe, 通知, key 过期等特性。

1447.最简分数

给你一个整数 n ，请你返回所有 0 到 1 之间（不包括 0 和 1）满足分母小于等于 n 的 最简 分数 。分数可以以 任意 顺序返回。

输出

分析

  • 本题主要难点在于判断是否可以约分

解决方法

  • 判断分子分母的最大公约数是否为1

代码

  • 时间复杂度: 枚举分子分母的复杂度为O(n^2);欧几里得算法:O(log^n);总时间复杂度O(n^2*log^n);
  • 空间复杂度：忽略递归带来的额外空间开销，复杂度为O(1)
